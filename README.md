# Global News Agent

Локальный новостной агент для поиска мировых новостей, новостей по тикеру, сырью/металлам и произвольным запросам.

## Запуск

```bash
npm install
npm start
```

Откройте `http://localhost:3000`.

Для генерации пары VAPID-ключей под push:

```bash
npm run generate:vapid
```

## Что умеет

- автоматически показывает новости за последние 24 часа: политика + обычные новости, только политика, обычные новости, экономика или технологии;
- ищет новости по тикеру компании, например `NVDA`, `AAPL`, `TSLA`;
- дает готовые варианты для меди, золота, серебра, нефти, газа, пшеницы, лития и урана;
- ранжирует результаты по смыслу, а не только по слову из запроса;
- объединяет похожие публикации в истории и показывает несколько источников внутри одного блока;
- сохраняет закрепленные запросы, историю поиска и алерты;
- умеет браузерные и push-уведомления по алертам;
- дает короткую сводку по текущей выборке;
- позволяет фильтровать по стране, языку, теме и типу источника;
- обновляет текущий поиск каждые 5, 15 или 30 минут;
- работает без платных ключей через Google News RSS, GDELT DOC API и прямые RSS-ленты популярных сайтов: Yahoo Finance, Investing.com, Benzinga, CNBC, MarketWatch, NYTimes, NPR и Fox News.

GDELT ограничивает частоту запросов, поэтому сервер добавляет кеш и очередь.

## Render

В репозитории уже есть [render.yaml](</Users/viktoria/Documents/New project 2/render.yaml>) для схемы `GitHub -> Render`.

- сервис поднимается как Node web service;
- health check идет на `/health`;
- данные приложения пишутся в `NEWS_AGENT_DATA_DIR`;
- push-уведомления используют `WEB_PUSH_PUBLIC_KEY`, `WEB_PUSH_PRIVATE_KEY` и `WEB_PUSH_SUBJECT`;
- для постоянного хранения истории, закреплений и алертов подключен persistent disk.

Важно: на Render локальная файловая система по умолчанию временная, поэтому для прод-версии дисковое хранилище здесь не декоративное, а нужное.

## Push в проде

Для полноценных push-уведомлений на телефоне и компьютере нужны:

1. HTTPS-домен
2. VAPID-ключи
3. Разрешение на уведомления у пользователя

Минимальный путь:

```bash
npm run generate:vapid
```

Потом перенесите значения в переменные Render:

- `WEB_PUSH_PUBLIC_KEY`
- `WEB_PUSH_PRIVATE_KEY`
- `WEB_PUSH_SUBJECT`

`WEB_PUSH_SUBJECT` обычно выглядит как `mailto:you@example.com`.

## GitHub -> Render -> домен

Сейчас у локального репозитория нет удаленного `origin`, поэтому финальный деплой в Render нельзя завершить полностью из этой папки без подключения GitHub.

Нужный порядок:

1. создать репозиторий на GitHub;
2. привязать его как `origin`;
3. запушить проект;
4. в Render создать Blueprint из этого репозитория;
5. ввести `sync: false` переменные для push;
6. подключить свой домен в Render Dashboard и подтвердить DNS.

После этого сайт будет доступен по HTTPS, и push-уведомления смогут приходить на телефон и компьютер уже как нормальные web push, а не только как локальные уведомления открытой вкладки.
